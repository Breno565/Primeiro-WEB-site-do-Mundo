O projeto WWW mescla as técnicas de recuperação de informações e hipertexto para criar um sistema de informações global fácil, mas poderoso. O projeto é baseado na filosofia de que muitas informações acadêmicas devem estar disponíveis gratuitamente para qualquer pessoa. O objetivo é permitir o compartilhamento de informações em equipes internacionalmente dispersas e a disseminação de informações por grupos de apoio. Originalmente voltado para a comunidade de física de alta energia, ele se espalhou para outras áreas e atraiu muito interesse em suporte ao usuário, descoberta de recursos e áreas de trabalho colaborativo.

Visualização do leitor

O mundo da WWW consiste em documentos e links. Índices são documentos especiais que, em vez de serem lidos, podem ser pesquisados. O resultado dessa pesquisa é outro documento ("virtual") que contém links para os documentos encontrados. Um protocolo simples ("HTTP") é usado para permitir que um programa de navegador solicite uma pesquisa por palavra-chave por um servidor de informações remoto. A web contém documentos em vários formatos. Os documentos hipertextos (reais ou virtuais) contêm links para outros documentos ou locais dentro dos documentos. Todos os documentos, reais, virtuais ou índices, parecem semelhantes ao leitor e estão contidos no mesmo esquema de endereçamento.

Para seguir um link, o leitor clica com o mouse (ou digita um número se não tiver o mouse). Para pesquisar e indexar, um leitor fornece palavras-chave (ou outros critérios de pesquisa). Essas são as únicas operações necessárias para acessar o mundo inteiro dos dados.

Visualização do provedor de informações

Os navegadores WWW podem acessar muitos sistemas de dados existentes através de protocolos existentes (FTP, NNTP) ou via HTTP e um gateway. Dessa forma, a massa crítica de dados é rapidamente excedida e o uso crescente do sistema por leitores e fornecedores de informações se incentiva. Criar uma web é tão simples quanto escrever alguns arquivos SGML que apontam para os dados existentes. Torná-lo público envolve a execução do daemon FTP ou HTTP e a criação de pelo menos um link na web a partir de outro. De fato, qualquer arquivo disponível por FTP anônimo pode ser imediatamente vinculado a uma web. O pequeno esforço inicial foi projetado para permitir pequenas contribuições. No outro extremo da escala, grandes provedores de informações podem fornecer a um servidor HTTP indexação de texto completo ou de palavras-chave. Isso pode permitir o acesso a um grande banco de dados existente sem alterar a maneira como o banco de dados é gerenciado. Esses gateways já foram transformados nos sistemas VMS / Help da Digital, Universidade Técnica do "Hyper-G" de Graz e sistemas "WAIS" da Thinking Machine.

O modelo WWW supera as frustrantes incompatibilidades de formato de dados entre fornecedores e leitores, permitindo a negociação de formatos entre um navegador inteligente e um servidor inteligente. Isso deve fornecer uma base para a extensão em multimídia e permitir que aqueles que compartilham padrões de aplicativos façam uso total deles na Web.

Este resumo não descreve as muitas possibilidades interessantes abertas pelo projeto WWW, como o armazenamento em cache eficiente de documentos. a redução de cópias desatualizadas redundantes e o uso de daemons do conhecimento. Há mais informações na documentação do projeto on-line, incluindo algumas informações sobre hipertexto e muitas notas técnicas.

Tente

Você pode experimentar o navegador de modo de linha simples telnetando para info.cern.ch (nenhum usuário ou senha. No UK JANET, use o Gateway). Você também pode descobrir mais sobre a WWW dessa maneira. Este é o navegador menos sofisticado - lembre-se de que os orientados a janelas são muito mais inteligentes! É muito mais eficiente instalar um navegador em sua própria máquina. O navegador de modo de linha está atualmente disponível no formato de origem por FTP anônimo no nó info.cern.ch [atualmente 128.141.201.74] como:

  • /pub/www/src/WWWLineMode_v.vv.tar.Z.

(v.vv é o número da versão - obtenha o mais recente.)

Também está disponível um editor de hipertexto para o NeXT (WWWNeXTStepEditor_v.vv.tar.Z), o navegador ViolaWWW para X11 e um daemon de servidor esqueleto (WWWDaemon_v.vv.tar.Z).

A documentação pode ser lida usando www. Uma versão em texto sem formatação das instruções de instalação está incluída no arquivo tar! A documentação e os artigos imprimíveis (postscript) estão em / pub / www / doc.
